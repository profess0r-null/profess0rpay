# Security Policy

## Supported Versions

Currently, only the latest release is actively supported with security updates.

| Version | Supported          |
| ------- | ------------------ |
| 1.0.x   | :white_check_mark: |
| < 1.0   | :x:                |

## Reporting a Vulnerability

Please do not report security vulnerabilities through public GitHub issues.

Instead, please report them directly to the maintainers by emailing `profess0r.null.xyz@gmail.com`.

We will try to acknowledge your email within 48 hours, and we'll send a more detailed response within 5 days indicating the next steps in handling your report.

## Responsible Disclosure

If you believe you have discovered a vulnerability, please:
1. Contact us privately.
2. Give us a reasonable amount of time to fix the issue before publishing it.
3. Do not exploit the vulnerability for any reason other than testing.
